1. Getting Started

1.1 Overview

TeamForge IDE is a next-generation autonomous agentic development platform that integrates six specialized AI agents, a multi-provider AI chat system, a virtual file system (VFS), and intelligent workflow orchestration into a single cohesive environment. Unlike traditional IDEs, TeamForge delegates complex development tasks to AI agents that can plan, code, review, test, and deploy autonomously — while keeping you in full control.

At its core, TeamForge uses GLM-4 models from Zhipu AI (via the z-ai-web-dev-sdk) as the default AI provider. It also supports NVIDIA NIM (80+ free models) and any OpenAI-compatible API endpoint, giving you flexibility to choose the AI backbone that suits your needs.

1.2 First Steps

  1. Launch TeamForge IDE — Open the application in your browser or desktop client. The main interface loads with the editor, chat panel, and file explorer ready.
  2. Sign in to AI providers — Navigate to Settings to configure your AI provider. The default Z-AI (GLM-4) requires your Zhipu AI API key. Enter it to activate the AI agents and chat features.
  3. Create or open a project — Use Ctrl+N to create a new file, or open an existing project folder from the file explorer in the sidebar.
  4. Start a chat — Click the chat panel on the right side, type a message, and press Enter. The AI agent will respond based on your project context.
  5. Try a slash command — Type /help in the chat input to see all available commands, or /edit to request an AI-powered code edit.

1.3 Project Setup

TeamForge IDE works with any project structure. To set up a project:

  • Open existing project — Click the folder icon in the sidebar or use Ctrl+O to browse and open a directory. All files are loaded into the VFS.
  • Create from template — Use the /create_file command or manually create files. The VFS supports all standard file types including .js, .ts, .py, .rs, .go, .html, .css, and more.
  • Initialize with a framework — Ask any AI agent: "Set up a Next.js project with TypeScript and Tailwind CSS". The agent will scaffold the project, create necessary files, and configure the environment.
  • Version control — TeamForge integrates with Git. Use the /commit slash command or the terminal to manage version control operations.
Tip: The VFS (Virtual File System) is the backbone of TeamForge. All file operations happen in-memory first, giving AI agents instant access to your entire codebase without disk I/O bottlenecks. Changes are persisted to disk on save (Ctrl+S).

3. AI Agents

3.1 Agent Overview

TeamForge IDE deploys six specialized AI agents, each designed for a specific phase of the software development lifecycle. All agents are powered by GLM-4 models and can be configured to use NVIDIA NIM or OpenAI-compatible providers.

Atlas
Architect

Designs system architecture, creates technical specifications, and plans project structure. Best for high-level design decisions, technology selection, and breaking down complex features into implementable components.

Codey
Developer

Writes and modifies code based on specifications. Handles core implementation — from scaffolding new files to implementing complex algorithms. Understands project context through the VFS and follows existing patterns.

Prism
Reviewer

Performs code reviews, identifies bugs, security vulnerabilities, and style violations. Provides detailed feedback with line-level annotations and suggests specific fixes. Enforces coding standards across the codebase.

Flux
Tester

Generates and runs test suites, identifies edge cases, and ensures code quality. Writes unit, integration, and end-to-end tests. Can execute tests via the terminal and analyze failures to suggest targeted fixes.

Blaze
DevOps

Manages deployment pipelines, Docker configurations, CI/CD workflows, and infrastructure. Builds containers, pushes images, configures environments, and handles release management.

Nova
Project Manager

Coordinates tasks across agents, tracks progress, and manages timelines. Breaks down user stories, assigns work to the appropriate agent, monitors completion, and provides status reports.

3.2 Working with Agents

Assigning Tasks

You can direct tasks to specific agents in three ways:

  1. Direct mention in chat — Prefix your message with the agent name: @Codey Add input validation to the login form. The named agent picks up the task immediately.
  2. Agent panel — Click an agent in the sidebar Agent Panel to open a dedicated conversation with that agent.
  3. Auto-routing — Simply describe your task without specifying an agent. Nova (the PM agent) analyzes the request and routes it to the best agent. Example: Review the auth module for security issues auto-routes to Prism.

3.3 Agent Collaboration

For complex multi-step tasks, agents collaborate through Nova's orchestration. When you request "Build a user registration feature", Nova may: assign Atlas to design the registration flow and data schema; assign Codey to implement the frontend form and backend API; assign Flux to write and run tests; assign Prism to review the final code before merge. Each agent's output is visible in the chat panel, and you can intervene at any step.

Tip: In YOLO Mode, agents execute tasks without waiting for your approval between steps. Ideal for rapid prototyping but use with caution in production codebases.

4. AI Chat System

4.1 Chat Interface & Model Selection

The AI Chat is your primary interaction point with TeamForge's intelligence layer. It supports multi-turn conversations, code-aware context, and real-time streaming responses. The model selector at the top of the chat panel lets you switch between providers mid-conversation.

Supported AI Providers

ProviderModelsSetupNotes
Z-AI (GLM-4) GLM-4, GLM-4-Plus, GLM-4-Flash API key in Settings Default provider. Best agent integration. Powered by Zhipu AI's z-ai-web-dev-sdk. GLM-4-Flash for quick tasks.
NVIDIA NIM 80+ free models (Llama, Mistral, Qwen, etc.) NVIDIA API key Free tier available. Great for experimentation. Select from dropdown after configuring your key.

How to Switch Models

  1. Click the model selector dropdown at the top of the chat panel (shows the currently active model name).
  2. Select a provider tab: Z-AI, NVIDIA NIM, or Custom.
  3. Choose a specific model from the list. The chat instantly switches — no page reload needed.
  4. Your conversation context is preserved when switching. The new model receives the full chat history.

4.2 Chat History & Session Management

TeamForge maintains a complete history of all chat sessions, organized by project and date. You can resume previous conversations or start fresh ones at any time.

  • New session — Click the "+" button in the chat header or press Ctrl+Shift+N. Starts a clean conversation with no prior context.
  • Resume a session — Click the clock icon in the chat header to view recent sessions. Select one to restore the full conversation history.
  • Pin important sessions — Right-click any session and select "Pin" to bookmark it. Pinned sessions appear at the top of the history list.
  • Delete sessions — Right-click and select "Delete" to remove a session permanently.

Context Awareness

The chat system automatically provides AI agents with relevant project context: open files in the editor are included in the context window; selected code is auto-referenced (also use @filename syntax); project structure from the VFS tree; and terminal output (errors, logs) when relevant.

Tip: For best results, keep relevant files open in the editor while chatting. This gives the AI maximum context. Use @filename to explicitly reference any project file in the VFS.

6. YOLO Mode

What is YOLO Mode?

YOLO Mode is TeamForge's autonomous execution mode. When enabled, AI agents operate without confirmation prompts — they can modify files, execute terminal commands, run tests, and make project changes directly. The name reflects the "You Only Live Once" philosophy: maximum speed, minimal friction.

6.1 How to Enable

  1. Click the YOLO toggle switch in the top bar. It turns green when active.
  2. Alternatively, type /yolo in the chat to toggle the mode.
  3. A confirmation dialog appears the first time you enable it, explaining the implications. Accept to proceed.

6.2 What Changes

7. File Management & VFS

7.1 Virtual File System (VFS)

The VFS is TeamForge's in-memory file layer that provides instant access to all project files for both you and the AI agents. It operates transparently — you interact with files normally, and the VFS handles synchronization between memory and disk.

  • Read — Files are loaded into the VFS when opened or referenced. The VFS caches content for fast re-access. Agents use VFS reads to understand your codebase.
  • Write — When you or an agent edit a file, the VFS updates in memory immediately. The editor reflects changes instantly. Press Ctrl+S to persist to disk.
  • Search — The VFS supports full-text and regex search across all files. The /search command uses this for semantic search.
  • Watch — The VFS watches for external file changes (e.g., from Git operations) and syncs automatically.

7.2 Creating & Editing Files

  • New file: Ctrl+N opens a new untitled file. Save with Ctrl+S to name it and persist.
  • AI-created files: Use /create_file path/to/file.ext to have an agent generate a file with appropriate boilerplate.
  • AI edits: Use /edit or @Codey to request modifications. In normal mode, edits show as a diff preview before applying.
  • Multi-file edits: Agents can modify multiple files in a single operation, maintaining consistency across related changes.

8. Terminal & Build

8.1 Integrated Terminal

TeamForge includes a fully functional terminal at the bottom of the interface. Toggle with Ctrl+`. The terminal supports multiple sessions — click "+" to open additional instances. AI agents can execute commands when authorized (or automatically in YOLO Mode).

  • Manual execution — Type any shell command and press Enter. Supports bash, zsh, and PowerShell.
  • AI-driven execution — Use /run <command> to have an agent execute and monitor a command.
  • Build commands — Trigger via /run (e.g., /run npm run build) or directly in the terminal.

8.2 Build & Deploy Workflow

  1. Build — Run /run npm run build (or equivalent). Blaze can detect your project type and suggest the correct build command.
  2. Test — Run /run_tests to execute the test suite. Flux analyzes failures and suggests fixes.
  3. Review — Run @Prism Review the latest changes for a quality check before deploying.
  4. Commit — Run /commit to stage and commit with an AI-generated message.
  5. Deploy — Run /deploy and Blaze handles the rest: containerization, image push, and deployment.

9. Settings & Configuration

9.1 AI Provider Configuration

Open Settings via the gear icon in the top bar or Ctrl+,. The Settings panel is organized into tabs for each provider.

Z-AI (GLM-4) — Default Provider

  1. Open Settings and navigate to the AI Providers tab.
  2. Under Z-AI, enter your Zhipu AI API key in the designated field.
  3. Select a default model: GLM-4 (balanced), GLM-4-Plus (highest quality), or GLM-4-Flash (fastest).
  4. Click Test Connection to verify your key. A green checkmark confirms connectivity.

NVIDIA NIM Setup

  1. In Settings, select the NVIDIA NIM tab.
  2. Enter your NVIDIA API key (get a free key at build.nvidia.com).
  3. Browse the model catalog (80+ models) and select your preferred model.
  4. Click Test Connection. NVIDIA NIM models are served via inference microservices.

OpenAI-Compatible Setup

  1. In Settings, select the Custom / OpenAI-Compatible tab.
  2. Enter the Base URL (e.g., https://api.your-provider.com/v1).
  3. Enter your API Key for the endpoint.
  4. Specify the Model ID (e.g., meta-llama/llama-3-70b).
  5. Optionally adjust: temperature, max tokens, top_p.
  6. Click Test Connection to verify the endpoint responds correctly.

9.2 Theme & Preferences

  • Theme toggle — Switch between Light and Dark themes via the top bar icon or Ctrl+Shift+T.
  • Font size — Adjust editor font size in Settings under Editor. Default: 14px.
  • Auto-save — Enable/disable automatic file saving. When on, files save on focus change.
  • Terminal shell — Choose your preferred shell (bash, zsh, PowerShell) under Terminal settings.

11. Troubleshooting

IssuePossible CauseSolution
AI agents not responding API key missing, invalid, or expired Open Settings and verify your API key under the active provider. Click "Test Connection" to check. If using Z-AI, ensure your Zhipu AI account has sufficient credits.
Slow AI responses Large context window or high-demand model Switch to GLM-4-Flash for faster responses. Close unnecessary files to reduce context size. For NVIDIA NIM, try a smaller model (e.g., Llama 3 8B instead of 70B).
Agent edits not applying Normal mode requires manual approval Check the chat panel for a pending diff preview. Click "Apply" to accept changes, or enable YOLO Mode for automatic application.
NVIDIA NIM connection fails Invalid API key or network issue Verify your NVIDIA API key at build.nvidia.com. Check firewall settings. Ensure the endpoint URL is correct in your configuration.
Custom endpoint not working Incorrect Base URL or format mismatch Ensure the Base URL ends with /v1 and follows the OpenAI chat completions format. Test with curl first to confirm the endpoint is reachable.
File changes not persisting Changes saved in VFS but not to disk Press Ctrl+S to save the current file. Use Ctrl+Shift+S to save all files. Enable auto-save in Settings.
Terminal not responding Process hung or terminal session crashed Click the trash icon to terminate the current process. Open a new terminal session with the "+" button. Check if a long-running process is blocking.
Chat history missing Session expired or storage cleared Check the chat history panel (clock icon). Pinned sessions persist longer. Consider exporting important conversations.
Need more help? If you encounter an issue not listed here, try using the /help command in the chat, or describe your problem to Nova (the PM agent) who can guide you through diagnostics and connect you with the right agent for resolution.

11.2 Performance Tips

  • Minimize open files — Each open file adds to the AI context window. Close files you don't need for the current task to reduce token usage and improve response speed.
  • Use GLM-4-Flash for simple tasks — Quick edits, small fixes, and short questions don't need the full GLM-4 model. Switch to Flash for 3-5x faster responses.
  • Be specific in agent requests — Instead of @Codey Fix the bug, say @Codey Fix the null reference error in UserService.ts line 42. Specificity reduces agent iteration cycles.
  • Use YOLO Mode judiciously — Enable YOLO Mode for well-defined tasks on feature branches. Disable it when exploring or when changes affect shared code.
  • Regular commits — Use /commit frequently to create checkpoints. This makes it easy to revert if an agent makes unintended changes.
